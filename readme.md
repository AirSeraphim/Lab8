# 📦 Задание: Система учёта заявок в сервисный центр ORM

## 🎯 Цель проекта

Разработать консольное Python-приложение с использованием SQLite для управления заявками на ремонт техники. Для работы с базой данных использовать модуль sqlalchemy

Работа должна включать:

* создание моделей данных,
* работу с отношениями между сущностями,
* выполнение CRUD-операций через ORM,
* покрытие логики функциями,
* прохождение автоматических тестов (CI).

---

## В рамках работы запрещено использовать прямые SQL-запросы (cursor.execute). 
## Вся работа с базой данных должна выполняться через ORM SQLAlchemy.

---

# 🧱 Требования к системе

## Сущности базы данных

### Таблица `Clients`

* `id`          INTEGER PRIMARY KEY
* `name`        STRING NOT NULL
* `phone`       STRING UNIQUE NOT NULL

### Таблица `Requests`

* `id` — INTEGER PRIMARY KEY
* `client_id` — INTEGER (FK → Clients.id)
* `device` — STRING NOT NULL
* `problem` — STRING NOT NULL
* `status` — STRING NOT NULL (значения: `new`, `in_progress`, `done`)
* `created_at` — DATE NOT NULL

---

### Связь между таблицами

* Client (1) → (N) Request
* один клиент может иметь много заявок

# 🧪 Требования к автоматической проверке (CI)

Система CI проверяет следующие аспекты:

## 🔹 1. База данных

* таблицы `Clients` и `Requests` создаются корректно
* настроены связи между таблицами

## 🔹 2. Клиенты

* клиент успешно добавляется
* телефон уникален
* повторная вставка вызывает IntegrityError

## 🔹 3. Заявки

* заявка создаётся корректно
* статус по умолчанию = `new`
* корректная связь client_id

## 🔹 4. Статусы

* статус обновляется
* недопустимый статус вызывает ошибку

## 🔹 5. Поиск

* поиск работает по device
* поиск работает по problem
* поддерживается частичное совпадение
